## Technical Decisions

### 1. Human-Verified Smoke Checklist

**Decision:** Use manual browser testing for smoke validation rather than full E2E automation.

**Rationale:**
- Visual/functional validation requires human judgment
- 27 items can be validated faster manually than writing E2E tests
- Automated unit/integration tests (14 tests from Plan 01) provide regression safety net
- Manual testing catches UX issues automated tests might miss

**Trade-offs:**
- ➕ Faster for one-time v1 release validation
- ➕ Catches visual/interaction issues
- ➖ Not repeatable without human effort
- ➖ Future releases would benefit from E2E automation

### 2. Zero-Defect Release Threshold

**Decision:** All BLOCKER/CRITICAL issues must be resolved; defer HIGH/MEDIUM/LOW to backlog.

**Rationale:**
- Personal-use v1 scope allows shipping with minor polish gaps
- Focus on correctness over perfection
- Documented deferred items provide backlog for v1.1

**Result:** Zero defects found during hardening (100% quality baseline achieved)
