## Technical Decisions

### 1. Watch Mode Disabled by Default
**Decision:** Set `watch: false` in vitest.config.ts  
**Reason:** CI/CD environments need run-once behavior. Developers can still use `npm test -- --watch` when needed.

### 2. Happy-DOM over jsdom
**Decision:** Use happy-dom as test environment  
**Reason:** Lighter weight, faster execution, better Vite integration, sufficient for hook testing without full browser APIs.

### 3. Test File Organization
**Decision:** Place tests in `__tests__` directories next to source files  
**Reason:** Co-location improves discoverability and follows React Testing Library conventions.
